Liquid Flow overview

Liquid Flow is a private, non-custodial, multichain crypto-payment layer that platforms embed. It gives you three systems — merchant payments, private fundraising, and automated payroll — behind one API, without ever holding your keys or your funds.

⚠️ Testnet. Everything described here runs on public testnets and is unaudited. Do not move real funds. An independent cryptographic and smart-contract audit gates mainnet — see the Roadmap and Security.

The non-custodial principle

This is the single rule the whole product is built around:

🔒 Liquid Flow can time and authorize, but never move funds. Your spend key is shown to you once and is never stored on our servers. If Liquid Flow were fully compromised, no user money could be taken.
  • Payments & fundraising: deposits land at one-time stealth addresses only you can sweep, using a key we never hold.
  • Payroll: funds sit in a contract you own; our trigger key can only release a company-defined payout on or after its scheduled time, to the recipient you set.

How a payment works

  1. Request — your platform calls POST /api/payments; we derive a fresh one-time stealth address.
  2. Pay — the payer sends crypto to that address on any supported chain. Nothing links it back to them.
  3. Confirm — our watcher sees the deposit on-chain and marks it settled (also on-demand when you poll).
  4. Notify — an HMAC-signed webhook hits your app with a copy-proof receipt; you release the goods.

Quickstart

Create a merchant, then a payment. Full details in the API Reference.

# 1. Create a merchant (returns an API key)
curl -X POST https://liquidflow-io.vercel.app/api/merchants \
  -H "Content-Type: application/json" \
  -d '{"name":"Acme","mode":"stealth","webhook":"https://acme.com/hook"}'

# 2. Create a payment (use the api_key from step 1)
curl -X POST https://liquidflow-io.vercel.app/api/payments \
  -H "Authorization: Bearer lf_live_..." \
  -H "Content-Type: application/json" \
  -d '{"amount":"1000000000000000","asset":"ETH","chain":"eip155:84532"}'
# → { "deposit_address": "0x…", "status": "awaiting_payment", … }
