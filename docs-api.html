API Reference

REST API

A small JSON/REST API. All amounts are in the asset's smallest base units (wei for 18-dp assets, e.g. 1000000 = 1 USDC at 6 dp). All timestamps are Unix ms unless noted.

Base URL & conventions

  • Base URL — https://liquidflow-io.vercel.app
  • Auth — merchant/payment endpoints take Authorization: Bearer <api_key>
  • Chains — eip155:84532 (Base), eip155:5042002 (Arc), solana, sui
  • Decimals — ETH 18 · USDC 6 · SOL/SUI 9. Send amount in base units.
⚠️ Testnet, unaudited. Keys and receipts here guard payment metadata only — never funds.

Merchants

POST /api/merchants

Create a merchant and receive an API key. In stealth mode you also get two spend keys (EVM + ed25519) shown once — they sweep your deposits.

curl -X POST /api/merchants -H "Content-Type: application/json" -d '{
  "name": "Acme",
  "mode": "stealth",            // "stealth" (private) or "instant"
  "plan": "Growth ($18/mo)",
  "chains": ["eip155:84532"],
  "settle": "USDC",
  "payout": "0x…",              // instant mode payout wallet
  "webhook": "https://acme.com/lf/hook"
}'
// → { "merchant_id","api_key":"lf_live_…","webhook_secret":"whsec_…",
//     "mode","plan","spend_key","spend_key_ed25519" }

GET /api/merchants

Authenticated. Returns the merchant profile (name, mode, plan, chains, payout, webhook secret, created_at).

Payments

POST /api/payments

Authenticated. Creates a payment and derives a one-time deposit address (stealth) for the chain.

curl -X POST /api/payments -H "Authorization: Bearer lf_live_…" \
  -H "Content-Type: application/json" -d '{
  "amount": "1000000000000000",  // base units (0.001 ETH)
  "asset": "ETH",
  "chain": "eip155:84532",
  "label": "Order #1024"
}'
// → { "payment_id","deposit_address","amount","asset","chain",
//     "decimals","symbol","status":"awaiting_payment","expires_at" }

GET /api/payments

Authenticated. Lists the merchant's payments plus stats. Confirms any pending deposit that has landed on-chain (on-demand).

// → { "payments":[…], "stats":{"total","confirmed","pending","volume"} }

GET /api/payments/:id

Returns one payment's status and runs an on-demand on-chain check (so a real deposit confirms within seconds). Merchant payments require the owning key; public fundraiser donations are readable by id.

POST /api/simulate/:id

Test-only. Marks a payment confirmed and fires its webhook — handy for testing your webhook handler without sending funds. → { "ok":true, "webhook_sent":true }

Fundraisers (Potlock)

POST /api/fundraisers

Create a private fundraising campaign with its own stealth keypair. Returns a shareable URL and the sweep key (once).

curl -X POST /api/fundraisers -H "Content-Type: application/json" -d '{
  "title": "Rebuild the library", "description":"…",
  "goal": 2, "chain": "eip155:84532"
}'
// → { "id","slug","url":"/potlock-private.html?id=…","spend_key" }

GET /api/fundraisers

Public list of campaigns with live totals (computed from confirmed on-chain donations).

GET /api/fundraisers/:id

Campaign details: raised, pct, donation_count, recent[], symbol, decimals.

POST /api/fundraisers/:id

Make a private donation. Body { "amount": "<base units>" } → returns a one-time deposit_address.

Payroll

The business deploys its own PayrollScheduler (native) or PayrollSchedulerERC20 (USDC) contract from its wallet, then registers it so Liquid Flow's trigger key can automate due releases. See Security for the non-custodial guarantee.

POST /api/payroll

curl -X POST /api/payroll -H "Content-Type: application/json" -d '{
  "contract": "0x…",  "company": "0x…", "name": "Acme payroll",
  "token": "0x1c7D…7238", "decimals": 6, "symbol": "USDC"   // omit token for native
}'

GET /api/payroll?company=0x…

List a company's payroll contracts.

GET /api/payroll/:contract

Reads the contract's payouts + balances from chain and runs the on-demand keeper — releasing any due, company-defined payouts via the trigger key (batched for ERC-20). Add ?readonly=1 to read without triggering.

Webhooks

When a payment confirms, Liquid Flow POSTs a JSON event to your webhook URL:

{ "type":"payment.confirmed", "payment_id":"pay_…", "amount":"…",
  "asset":"ETH", "chain":"eip155:84532", "confirmations":3, "final":true }

Each request carries a signature header. Verify it with your webhook_secret:

Header:  LF-Signature: t=<ms>,v1=<hex>
signed = HMAC_SHA256(webhook_secret, `${t}.${rawBody}`)   // compare to v1

// Node
const [ts,v1] = header.split(',').map(s=>s.split('=')[1]);
const expected = crypto.createHmac('sha256', secret).update(`${ts}.${raw}`).digest('hex');
const ok = crypto.timingSafeEqual(Buffer.from(v1), Buffer.from(expected));

Reject if the signature fails or the timestamp is stale. A copy-proof receipt is also issued to the payer as a fallback if a webhook is missed.
